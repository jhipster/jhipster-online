Continuous Integration

The following tools are supported:

GitHub Actions

  • Warning! as GitHub recently updated its OAuth2 token policies, you might need to go to the Configure screen to refresh your token and give "workflow" access to JHipster Online
  • A github-ci.yml will be added to your project.
  • Push the project to GitHub; the actions workflow will be configured on your GitHub repo.

Jenkins

  • A Jenkinsfile will be added to your project.
  • Go to your Jenkins instance
  • Create a new job, using a `Pipeline script from SCM`
  • Point the SCM configuration to your GitHub or GitLab repository

Gitlab

  • A .gitlab-ci.yml will be added to your project.
  • Your PR now is running using the GitLab Continuous Integration

Azure Pipelines

  • A azure-pipelines.yml will be added to your project.
  • Go to your project in your Azure DevOps organization.
  • Create a new pipeline using your project

Travis

  • A .travis.yml will be added to your project.
  • Go to your Travis profile page.
  • Click on "Sync account" on your Travis profile page.
  • Enable your project on your Travis profile page.

CircleCI

  • A circle.yml will be added to your project.
  • Create a CircleCI integration depending on your VCS; https://circleci.com/integrations/
